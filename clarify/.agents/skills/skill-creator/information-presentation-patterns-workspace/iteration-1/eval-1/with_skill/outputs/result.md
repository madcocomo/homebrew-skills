# 电商订单处理系统代码分析

### 一、系统整体架构

该系统包含两个核心类：
1. **Order（订单类）** - 负责单个订单的状态管理和业务逻辑
2. **OrderProcessor（订单处理器）** - 负责订单的创建和管理

---

### 二、订单状态定义（OrderStatus）

系统定义了6种订单状态：
- `PENDING`（待支付）- 订单创建后的初始状态
- `PAID`（已支付）- 客户完成支付后
- `SHIPPED`（已发货）- 商家发货后
- `COMPLETED`（已完成）- 客户确认收货后
- `CANCELLED`（已取消）- 订单被取消
- `REFUNDED`（已退款）- 支付被退回

---

### 三、Order类核心功能

#### 1. 订单创建（`__init__`）
```python
def __init__(self, order_id, amount, customer_type="normal"):
```
- 接受订单ID、订单金额、客户类型（普通用户或VIP用户）
- 初始状态设为 `PENDING`（待支付）

#### 2. 支付功能（`pay`）
- **前置条件**：只能是待支付状态的订单才能支付
- **折扣计算**：根据客户类型和订单金额自动计算折扣
- **返回**：返回实际支付金额（订单金额 - 折扣）

#### 3. 折扣计算逻辑（`calculate_discount`）
```
VIP客户：
  - 订单金额 >= 1000: 享85折（15%折扣）
  - 订单金额 >= 500: 享9折（10%折扣）

普通客户：
  - 订单金额 >= 1000: 享9折（10%折扣）
```

#### 4. 发货功能（`ship`）
- **前置条件**：只能是已支付状态的订单才能发货

#### 5. 订单完成（`complete`）
- **前置条件**：只能是已发货状态的订单才能确认完成

#### 6. 取消订单（`cancel`）
- **限制**：已发货或已完成的订单不能取消
- 取消后订单状态变为 `CANCELLED`

#### 7. 退款功能（`refund`）
- **前置条件**：只能是已支付状态的订单才能退款
- 退款后订单状态变为 `REFUNDED`

---

### 四、OrderProcessor类功能

#### 1. 创建订单（`create_order`）
- 检查订单ID是否已存在（防止重复创建）
- 创建新的Order实例并存入orders字典

#### 2. 处理订单（`process_order`）
支持5种操作：
- `pay` - 支付订单
- `ship` - 发货
- `complete` - 确认完成
- `cancel` - 取消订单
- `refund` - 退款

---

### 五、完整工作流程

```
创建订单 → 支付（计算折扣） → 发货 → 确认完成
                    ↓
              [可选：退款]
                    ↓
              [可选：取消订单]
```

**典型业务流程**：
1. 调用 `create_order()` 创建订单 → 状态：PENDING
2. 调用 `pay()` 支付订单 → 状态：PAID
3. 调用 `ship()` 发货 → 状态：SHIPPED
4. 调用 `complete()` 确认完成 → 状态：COMPLETED

---

### 六、关键设计特点

1. **状态机模式**：每个操作都有严格的状态检查，确保业务逻辑的严谨性
2. **折扣策略**：根据客户等级和消费金额动态计算折扣，激励消费
3. **异常处理**：关键操作都有状态验证，防止非法操作（如已发货的订单不能取消）
