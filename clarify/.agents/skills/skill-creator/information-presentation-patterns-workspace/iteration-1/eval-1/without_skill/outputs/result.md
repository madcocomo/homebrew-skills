# 电商订单处理系统代码分析

## 代码结构概述

这段代码包含三个主要的类，实现了完整的订单生命周期管理：

### 1. OrderStatus（订单状态枚举类）

定义了订单的6种状态：
- **PENDING**（待支付）：订单创建后的初始状态
- **PAID**（已支付）：客户完成支付后的状态
- **SHIPPED**（已发货）：商家已发货的状态
- **COMPLETED**（已完成）：客户确认收货后的最终状态
- **CANCELLED**（已取消）：订单被取消的状态
- **REFUNDED**（已退款）：支付被退回的状态

### 2. Order（订单类）

这是订单的核心实体类，包含以下属性和方法：

**属性：**
- `order_id`：订单唯一标识
- `amount`：订单金额
- `customer_type`：客户类型（normal普通用户 或 vipVIP用户）
- `status`：当前订单状态，初始为PENDING

**核心方法：**

- **pay()（支付方法）**
  - 验证订单状态必须是PENDING才能支付
  - 调用calculate_discount()计算折扣
  - 将订单状态改为PAID
  - 返回实际支付金额（订单金额减去折扣）

- **calculate_discount()（折扣计算方法）**
  - VIP用户：
    - 订单金额>=1000：享85折（15%折扣）
    - 订单金额>=500：享9折（10%折扣）
  - 普通用户：
    - 订单金额>=1000：享9折（10%折扣）
  - 其他情况：无折扣

- **ship()（发货方法）**
  - 验证订单状态必须是PAID才能发货
  - 将状态改为SHIPPED

- **complete()（完成方法）**
  - 验证订单状态必须是SHIPPED才能确认完成
  - 将状态改为COMPLETED

- **cancel()（取消方法）**
  - 验证订单状态不是SHIPPED或COMPLETED才能取消
  - 将状态改为CANCELLED
  - 已发货或已完成的订单不能取消

- **refund()（退款方法）**
  - 验证订单状态必须是PAID才能退款
  - 将状态改为REFUNDED

### 3. OrderProcessor（订单处理器类）

负责管理所有订单的创建和处理：

**属性：**
- `orders`：字典，存储所有订单，键为order_id，值为Order对象

**核心方法：**

- **create_order()（创建订单）**
  - 验证订单ID不存在，避免重复创建
  - 创建新的Order对象并存储到字典中
  - 返回创建的订单对象

- **process_order()（处理订单）**
  - 根据order_id获取订单对象
  - 根据action参数执行相应操作：
    - "pay"：支付订单
    - "ship"：发货
    - "complete"：完成订单
    - "cancel"：取消订单
    - "refund"：退款

## 工作流程

1. **创建订单**：通过OrderProcessor创建新订单，订单状态为PENDING
2. **支付订单**：调用pay()方法，根据客户类型和金额计算折扣，更新状态为PAID
3. **发货**：商家调用ship()方法，将状态更新为SHIPPED
4. **确认收货**：客户调用complete()方法，状态更新为COMPLETED
5. **取消订单**：在发货前可取消订单
6. **退款**：支付后可申请退款

## 设计特点

- **状态机模式**：使用枚举定义清晰的状态转换
- **状态验证**：每个操作都验证当前状态是否允许执行
- **折扣策略**：根据客户等级和消费金额应用不同折扣
- **错误处理**：使用ValueError抛出异常处理无效操作
