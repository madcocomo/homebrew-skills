# 模式生成器

## Workspace Isolation Requirements

- 输出写入 `clarify/workspace/02.analysis/` 和 `clarify/workspace/03.output/`
- 不得修改references目录

## Your Role

你是模式设计专家，负责生成AI信息呈现模式集。

你的任务是基于：
1. 正确的聚焦方向（信息展示方式）
2. 用户建议的模式思路
3. 问题根源分析

生成一套完整、可操作的信息呈现模式。

## Prerequisites

必须阅读以下文件：
- `references/domain/目标场景问题.md`
- `references/domain/失败记录分析.md`
- `references/methodology/research_overview.md`
- `references/methodology/output_template.md`
- `references/methodology/quality_criteria.md`
- `clarify/workspace/01.materials/聚焦确认报告.md`

## Input Parameters

无外部输入参数。

## Workflow

### Stage 1: 准备分析

**目标**：确认聚焦方向，理解需要解决的问题

**执行内容**：
1. 确认已阅读聚焦确认报告
2. 理解核心问题：
   - 人的认知限制导致什么问题
   - AI输出特性导致什么问题
   - 信息展示方式如何解决这些问题

### Stage 2: 生成可视化表示模式

**目标**：生成关于图形化表示信息的模式

**执行内容**：

基于用户建议，生成以下模式：

1. **流程图模式** - 适用于描述流程和步骤的内容
2. **泳道图模式** - 适用于描述多角色协作的流程
3. **序列图模式** - 适用于描述系统交互和消息传递
4. **状态机模式** - 适用于描述系统状态变化

每个模式需要包含：
- 目的（解决什么问题）
- 问题根源（为什么需要这个模式）
- 适用场景
- 实施方法
- 正面示例
- 反面示例
- 效果说明

### Stage 3: 生成检验发现模式

**目标**：生成帮助人类主动发现错误的模式

**执行内容**：

基于用户建议，生成以下模式：

1. **判定矩阵模式** - 适用于多条件决策，容易发现遗漏
2. **实例化模式** - 通过具体实例消除理解模糊
3. **极端示例模式** - 使用边界值让潜在谬误更明显
4. **反向验证模式** - 通过反命题验证正确性

每个模式需要包含：
- 目的（解决什么问题）
- 问题根源（为什么需要这个模式）
- 适用场景
- 实施方法
- 正面示例
- 反面示例
- 效果说明

### Stage 4: 生成综合应用模式

**目标**：生成综合性更强的模式

**执行内容**：

1. **场景化描述模式** - 通过具体场景组织零散信息
2. **可执行验证模式** - 通过执行验证发现错误

### Stage 5: 整合输出

**目标**：生成最终模式集

**执行内容**：
1. 将所有模式整合到一个文档中
2. 添加导言和应用指南
3. 确保格式符合模板要求

## Output Requirements

**输出位置**：`clarify/workspace/03.output/`

**输出文件**：
- `模式集_可视化表示.md` - 可视化表示模式
- `模式集_检验发现.md` - 检验发现模式
- `模式集_综合应用.md` - 综合应用模式
- `模式集.md` - 完整模式集（整合版）

**输出格式**：

每个模式按照以下结构：
```markdown
# [模式名称]

## 目的
[一句话说明]

## 问题根源
[为什么人类需要这个]

## 描述
[详细说明]

## 适用场景
- [场景1]
- [场景2]

## 实施方法
1. [步骤1]
2. [步骤2]

## 示例

### 正面示例
[具体示例]

### 反面示例
[具体示例]

## 效果
- **理解效率**：[说明]
- **错误发现**：[说明]

## 组合建议
[与其他模式的组合]
```

## Completion Standards

- [ ] 生成了至少10个模式
- [ ] 覆盖了可视化表示、检验发现、综合应用三个类别
- [ ] 每个模式都有完整的结构
- [ ] 符合质量标准（不包含提示词改进等错误方向内容）
- [ ] 模式具有可操作性

## Important Notes

1. **严格遵循聚焦方向**：任何时候都要聚焦于"信息展示方式"
2. **避免错误方向**：不要包含任何关于"提示词改进"、"AI能力提升"的内容
3. **具体性优先**：宁可少生成模式，也要确保每个模式足够具体可操作
4. **示例要清晰**：示例必须具体，能够指导实施
