# 质量标准

## 审查清单

### 1. 聚焦性审查

**每个模式必须满足：**

- [ ] 直接作用于"信息展示方式"
- [ ] 目的是帮助人类理解，而非帮助AI生成
- [ ] 目的是帮助人类发现错误，而非减少错误

**排除项（如果出现以下内容，模式不合格）：**
- [ ] 包含"如何改进提示词"的建议
- [ ] 包含"如何让AI更可靠"的建议
- [ ] 聚焦于AI能力提升而非展示方式优化

### 2. 具体性审查

- [ ] 每个模式有明确可执行的实施步骤
- [ ] 有具体的示例说明
- [ ] 示例包含正面和反面两种情况

### 3. 实用性审查

- [ ] 明确说明模式的适用场景
- [ ] 明确说明模式的效果（理解效率、错误发现）
- [ ] 提供模式组合使用的建议

### 4. 完整性审查

- [ ] 覆盖用户建议的主要模式类型
- [ ] 包含可视化表示类模式
- [ ] 包含检验发现类模式
- [ ] 包含综合应用类模式

## 质量检查问题

在生成每个模式时，问自己：

1. **这个模式是关于"如何展示"还是"如何生成"？**
2. **这个模式的目的是帮助人类还是帮助AI？**
3. **如果一个人类看到这个模式的输出，能否主动发现其中的错误？**
4. **这个模式是否足够具体，能够直接指导实施？**
