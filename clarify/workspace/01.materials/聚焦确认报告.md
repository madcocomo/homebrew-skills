# 聚焦确认报告

## 1. 核心目标

**产生一套模式集合，用于指导AI在展示信息时更容易让人理解和发现错误。**

## 2. 正确聚焦

### 2.1 聚焦什么

- **聚焦于信息展示方式**：关注AI输出内容如何被组织和呈现，而非AI如何生成内容
- **帮助人类理解**：使人类能够更高效地理解AI输出的信息
- **帮助人类发现错误**：使人类能够更容易发现AI输出中的错误或问题
- **降低认知负担**：通过优化信息结构帮助人类处理复杂信息
- **便于主动验证**：提供便于人类主动检查和校验的结构

### 2.2 不聚焦什么

- **不聚焦于提示词改进**：不研究如何写更好的提示词让AI生成更好结果
- **不聚焦于AI能力提升**：不研究如何让AI更可靠、更准确
- **不聚焦于减少错误**：不研究如何减少AI产生错误（这是AI的职责）
- **不聚焦于泛泛而谈的可解释性**：需要具体可操作的技术，而非抽象原则

## 3. 错误方向（必须避免）

| 错误方向 | 问题说明 |
|---------|---------|
| ❌ 提示词改进 | 聚焦于"如何让AI生成更好内容"，而非"如何让内容更容易理解" |
| ❌ 置信度标注 | 试图让AI"更可靠"，这是帮助AI而非帮助人类 |
| ❌ 边界识别 | 试图让AI"知道自己能力边界"，这是AI的职责 |
| ❌ 推理外显 | 试图让AI"展示思考过程"，这是改进AI生成方式 |
| ❌ 知识边界 | 试图让AI"承认不知道"，这是AI的自我改进 |
| ❌ 泛化可解释性 | 缺乏具体可操作的展示技术 |

## 4. 用户建议模式的评估

### 4.1 符合正确聚焦的

| 模式 | 原因 |
|-----|------|
| ✅ 流程图 | 信息展示方式，帮助人类理解流程结构 |
| ✅ 泳道图 | 信息展示方式，帮助人类理解多角色协作 |
| ✅ 序列图 | 信息展示方式，帮助人类理解系统交互 |
| ✅ 状态机 | 信息展示方式，帮助人类理解状态变化 |
| ✅ 判定矩阵 | 检验发现机制，帮助人类发现遗漏 |
| ✅ 实例化 | 检验发现机制，通过具体例子帮助人类理解抽象概念 |
| ✅ 极端示例 | 检验发现机制，帮助人类发现潜在谬误 |
| ✅ 反向验证 | 检验发现机制，帮助人类验证信息正确性 |
| ✅ 场景化描述 | 综合应用，帮助人类在具体场景中理解零散信息 |
| ✅ 可执行验证 | 综合应用，通过执行帮助人类发现错误 |

### 4.2 无需修正的

用户建议的所有10个模式方向都符合正确的聚焦方向。

## 5. 结论

**本系统生成模式时应聚焦于：**

- **具体的信息展示技术**：流程图、泳道图、序列图、状态机、判定矩阵等可视化表示
- **检验发现机制**：实例化、极端示例、反向验证、可执行验证等帮助人类主动发现错误
- **信息组织方式**：场景化描述等帮助人类理解零散信息

**必须避免：**

- 任何关于"改进提示词"的建议
- 任何关于"让AI更可靠"的建议
- 任何关于"减少AI错误"的建议
- 泛泛而谈的"可解释性"原则
