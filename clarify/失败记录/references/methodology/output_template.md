# 输出模板

## 文档结构

```markdown
# AI呈现清晰度模式集

## 一、概述
### 1.1 研究背景
### 1.2 研究目标
### 1.3 模式分类体系

## 二、可理解性增强模式
### 2.1 [模式名称]
    - 模式描述
    - 适用场景
    - 实施指南
    - 示例

### 2.2 [模式名称]
    ...

## 三、错误发现模式
### 3.1 [模式名称]
    - 模式描述
    - 适用场景
    - 实施指南
    - 示例

### 3.2 [模式名称]
    ...

## 四、综合应用指南
### 4.1 模式组合建议
### 4.2 场景适配指南

## 五、附录
### 5.1 术语表
### 5.2 参考资料
```

## 格式规范

### 标题层级

- 一级标题 (#): 文档标题，只有一个
- 二级标题 (##): 主要章节
- 三级标题 (###): 模式名称
- 四级标题 (####): 模式的子节

### 模式描述模板

```markdown
### [模式名称]

**分类**: [可理解性增强 / 错误发现]

**目的**: [简述模式要解决的问题]

**描述**:
[模式的详细描述，包括：
- 模式的核心理念
- 为什么有效
- 解决了什么问题]

**适用场景**:
- [场景1]
- [场景2]

**实施指南**:
1. [步骤1]
2. [步骤2]
3. [步骤3]

**示例**:

*正面示例*:
```markdown
[好的例子]
```

*反面示例*:
```markdown
[不好的例子]
```

**注意事项**:
- [注意点1]
- [注意点2]

**与其他模式的关系**:
- [关联模式1]: [关系描述]
- [关联模式2]: [关系描述]
```

### 引用格式

- 学术引用: `[作者, 年份]`
- 网页引用: `[标题](URL)`
- 模式内部引用: `详见[模式名称]`

### 列表规范

- 无序列表使用 `-` 或 `*`
- 有序列表使用 `1.`, `2.`, `3.`
- 列表项过长时使用缩进

## 写作风格

### 语言要求

- 使用简体中文
- 术语保持一致性
- 表达准确，避免歧义
- 学术性与实用性并重

### 表述规范

- 避免过度使用被动语态
- 例子要具体、贴近实际
- 抽象概念要配以实例说明
- 重要的边界条件要明确标注

### 质量检查清单

- [ ] 每个模式都有清晰的定义
- [ ] 每个模式都有具体的实施指南
- [ ] 每个模式都有正反例示
- [ ] 模式分类逻辑一致
- [ ] 章节结构层次清晰
- [ ] 术语使用保持一致
- [ ] 内容无逻辑矛盾
- [ ] 具有实际可操作性
