## 强化关系

### 强强化关系（组合使用效果显著）

| 模式A | 模式B | 关系 | 说明 |
|-------|-------|------|------|
| PAT-001 推理外显 | PAT-013 一致性验证 | 强化 | 外显推理过程便于验证一致性 |
| PAT-002 层次化解释 | PAT-003 信息密度控制 | 强化 | 层次化结构帮助控制信息密度 |
| PAT-006 边界标注 | PAT-014 知识边界识别 | 强化 | 边界识别是标注的实现基础 |
| PAT-010 置信度标注 | PAT-012 溯源标注 | 强化 | 置信度需要证据来源支撑 |
| PAT-004 结构化输出 | PAT-009 格式标准化 | 强化 | 结构化需要标准化格式支撑 |

### 中等强化关系

| 模式A | 模式B | 关系 | 说明 |
|-------|-------|------|------|
| PAT-001 推理外显 | PAT-002 层次化解释 | 强化 | 推理可分层级展示 |
| PAT-005 角色设定 | PAT-006 边界标注 | 强化 | 角色需要有边界 |
| PAT-008 反馈明确 | PAT-011 可操作错误 | 强化 | 错误消息是反馈的一种 |
| PAT-007 示例引导 | PAT-004 结构化输出 | 强化 | 示例可说明格式 |

## 典型模式组合

### 组合一：详细解释场景
**目标**: 提供详尽、可验证的解释

```
PAT-001 (推理外显)
    ↓ 强化
PAT-002 (层次化解释)
    ↓ 强化
PAT-003 (信息密度控制)
    ↓ 组合
PAT-010 (置信度标注) + PAT-012 (溯源标注)
```

**适用场景**: 学术研究、分析报告

### 组合二：代码生成场景
**目标**: 生成高质量、可维护的代码

```
PAT-004 (结构化输出)
    ↓ 强化
PAT-009 (格式标准化)
    ↓ 组合
PAT-011 (可操作错误消息)
    ↓ 组合
PAT-015 (自文档化模式)
```

**适用场景**: 代码编写、代码审查

### 组合三：问答对话场景
**目标**: 提供专业、可靠的答案

```
PAT-005 (角色设定)
    ↓ 组合
PAT-006 (边界标注) + PAT-014 (知识边界识别)
    ↓ 组合
PAT-010 (置信度标注) + PAT-007 (示例引导)
```

**适用场景**: 客服问答、咨询对话

### 组合四：高风险决策场景
**目标**: 确保信息可靠、可验证

```
PAT-010 (置信度标注)
    ↓ 组合
PAT-012 (溯源标注) + PAT-013 (一致性验证)
    ↓ 组合
PAT-006 (边界标注) + PAT-014 (知识边界识别)
```

**适用场景**: 医疗建议、法律咨询、金融分析
