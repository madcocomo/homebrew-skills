# 模式分析报告

本文档记录了从研究资料中识别的所有模式要素。

---

## 模式总览

共识别出 **14个核心模式**，其中：
- 可理解性增强模式：9个
- 错误发现模式：5个

---

## 一、可理解性增强模式

### PAT-001: 推理外显模式

**模式分类**: 可理解性增强 - 信息结构层

**核心要素**:
- 展示推理步骤而非仅给出结论
- 使用"一步步思考"引导
- 分阶段呈现推导过程

**来源资料**:
- SRC-006: Chain-of-Thought Prompting - 推理显式化
- SRC-008: Tree of Thoughts - 结构化思考
- SRC-007: Self-Consistency - 多路径推理

**实施要点**:
1. 在提示中加入推理引导词
2. 要求模型分步骤展示思考过程
3. 对复杂问题使用分支思考结构
4. 验证推理路径的一致性

**效果**:
- 正面：提升推理透明度，便于理解模型如何得出结论
- 负面：可能增加输出长度，需要权衡详细程度

**关联模式**:
- PAT-004: 层次化解释模式 - 推理与解释互补
- PAT-008: 一致性验证模式 - 验证推理正确性

---

### PAT-002: 层次化解释模式

**模式分类**: 可理解性增强 - 信息结构层

**核心要素**:
- 概览优先，细节按需
- 分层组织信息复杂度
- 支持渐进式深入

**来源资料**:
- SRC-013: Progressive Disclosure - 渐进式披露
- SRC-015: Visual Information Seeking - 概览优先
- SRC-001: XAI综述 - 局部与全局解释

**实施要点**:
1. 首先提供高层概述
2. 按重要性分层组织内容
3. 支持用户按需获取细节
4. 使用折叠/展开等交互机制

**效果**:
- 正面：减少认知负荷，适应不同需求
- 负面：可能遗漏关键细节

**关联模式**:
- PAT-001: 推理外显模式 - 层次化推理
- PAT-003: 信息密度控制模式 - 容量管理

---

### PAT-003: 信息密度控制模式

**模式分类**: 可理解性增强 - 表达策略层

**核心要素**:
- 符合工作记忆容量限制（7±2组块）
- 避免信息过载
- 适度分组和分段

**来源资料**:
- SRC-017: Working Memory - 7±2组块理论
- SRC-012: Cognitive Load Theory - 认知负荷优化

**实施要点**:
1. 单次呈现信息不超过7个要点
2. 使用列表、表格等分组方式
3. 适当留白，避免密集文本
4. 根据任务复杂度调整信息量

**效果**:
- 正面：提升信息吸收效率
- 负面：可能过度简化复杂内容

**关联模式**:
- PAT-002: 层次化解释模式 - 分层减少密度

---

### PAT-004: 结构化输出模式

**模式分类**: 可理解性增强 - 表达策略层

**核心要素**:
- 使用明确的数据格式（JSON、表格、列表）
- 保持格式一致性
- 支持机器解析和人工阅读

**来源资料**:
- SRC-010: Anthropic提示工程 - 结构化输出
- SRC-031: LangChain输出解析器 - 格式解析
- SRC-020: Code Complete - 代码可读性

**实施要点**:
1. 明确指定输出格式要求
2. 使用JSON Schema等定义结构
3. 保持标题层级清晰
4. 一致使用列表和表格

**效果**:
- 正面：提升可解析性，便于后续处理
- 负面：增加格式约束，可能限制灵活性

**关联模式**:
- PAT-009: 格式标准化模式 - 强化结构

## 二、错误发现模式

### PAT-010: 置信度标注模式

**模式分类**: 错误发现 - 表达策略层

**核心要素**:
- 明确标注置信度水平
- 区分高/中/低置信度
- 说明置信度判断依据

**来源资料**:
- SRC-024: Uncertainty Quantification - 不确定性量化
- SRC-025: Calibrated Language Model Evaluation - 校准评估
- SRC-028: Probability Calibration - 概率校准

**实施要点**:
1. 使用明确词语标注置信度
2. 区分事实性内容和推测性内容
3. 说明判断依据的可靠性
4. 定期评估置信度准确性

**效果**:
- 正面：帮助用户评估可靠性
- 负面：过度标注可能降低效率

**关联模式**:
- PAT-006: 边界标注模式 - 知识边界
- PAT-012: 溯源标注模式 - 证据来源

## 模式关系分析

### 相互增强的模式组合

1. **推理外显 + 一致性验证**: 推理过程显式化后，一致性检查更有效
2. **层次化解释 + 信息密度控制**: 层次化结构天然帮助控制密度
3. **置信度标注 + 溯源标注**: 置信度需要证据支撑
4. **边界标注 + 知识边界识别**: 边界识别是边界标注的实现

### 可能冲突的模式组合

1. **推理外显 vs 信息密度控制**: 详细推理可能增加信息密度
2. **结构化输出 vs 角色设定**: 过度结构化可能限制角色灵活性
3. **置信度标注 vs 角色设定**: 频繁质疑可能损害角色权威

### 模式适用边界

- **高风险场景**（医疗、法律、金融）: 优先使用 PAT-010, PAT-012, PAT-014
- **创意生成场景**: 优先使用 PAT-001, PAT-005, PAT-007
- **信息查询场景**: 优先使用 PAT-002, PAT-006, PAT-012
- **代码生成场景**: 优先使用 PAT-004, PAT-009, PAT-011
