## 模式总览

| 编号 | 模式名称 | 分类 | 功能 | 一句话描述 |
|------|----------|------|------|------------|
| PAT-001 | 推理外显模式 | 可理解性增强 | 信息结构 | 展示推理过程而非仅给结论 |
| PAT-002 | 层次化解释模式 | 可理解性增强 | 信息结构 | 从概要到细节逐层深入 |
| PAT-003 | 信息密度控制模式 | 可理解性增强 | 表达策略 | 控制信息量避免过载 |
| PAT-004 | 结构化输出模式 | 可理解性增强 | 表达策略 | 使用JSON/表格等格式组织 |
| PAT-005 | 角色设定模式 | 可理解性增强 | 交互反馈 | 明确AI扮演的专业角色 |
| PAT-006 | 边界标注模式 | 可理解性增强 | 表达策略 | 标注能力边界和适用范围 |
| PAT-007 | 示例引导模式 | 可理解性增强 | 表达策略 | 用示例说明期望输出 |
| PAT-008 | 反馈明确模式 | 可理解性增强 | 交互反馈 | 提供清晰的状态反馈 |
| PAT-009 | 格式标准化模式 | 可理解性增强 | 信息结构 | 统一格式规范 |
| PAT-010 | 置信度标注模式 | 错误发现 | 表达策略 | 标注信息可靠性等级 |
| PAT-011 | 可操作错误消息模式 | 错误发现 | 交互反馈 | 错误时提供解决建议 |
| PAT-012 | 溯源标注模式 | 错误发现 | 信息结构 | 标注信息来源便于验证 |
| PAT-013 | 一致性验证模式 | 错误发现 | 信息结构 | 检查并标注潜在矛盾 |
| PAT-014 | 知识边界识别模式 | 错误发现 | 表达策略 | 承认不确定和未知领域 |
| PAT-015 | 自文档化模式 | 通用 | 信息结构 | 输出包含自身说明 |

## 按功能快速查找

### 需要增强理解时

需要解释推理过程 → **PAT-001 推理外显**
需要分层次呈现 → **PAT-002 层次化解释**
需要控制信息量 → **PAT-003 信息密度控制**
需要格式规范 → **PAT-004 结构化输出** / **PAT-009 格式标准化**

### 需要发现错误时

需要标注可靠性 → **PAT-010 置信度标注**
需要验证来源 → **PAT-012 溯源标注**
需要检查一致 → **PAT-013 一致性验证**
需要识别不确定 → **PAT-014 知识边界识别**

### 改善交互时

需要明确角色 → **PAT-005 角色设定**
需要清晰边界 → **PAT-006 边界标注**
需要提供示例 → **PAT-007 示例引导**
需要状态反馈 → **PAT-008 反馈明确**

  ├─ 生成/创作类 → 需要格式? ──是──→ PAT-004 结构化输出

  │                        否

  │

  │                        │

  │                        ▼

                         │
                         否
