# 分析方法指南

## 模式分析框架

### 1. 结构完整性分析

对每个模式，分析其是否包含以下核心要素：

| 要素 | 描述 | 重要性 |
|------|------|--------|
| 模式名称 | 清晰、简洁的名称 | 必须 |
| 分类标签 | 功能分类和层次分类 | 必须 |
| 目的 | 模式要解决的问题 | 必须 |
| 描述 | 模式原理和核心概念 | 必须 |
| 适用场景 | 适用的具体场景 | 必须 |
| 实施指南 | 具体的实施步骤 | 必须 |
| 示例 | 正面和反面示例 | 必须 |
| 注意事项 | 潜在问题和边界情况 | 应该 |
| 关联关系 | 与其他模式的关系 | 应该 |

### 2. 质量评估标准

#### 清晰性评估
- 术语使用是否一致
- 描述是否无歧义
- 层次结构是否清晰

#### 可操作性评估
- 实施步骤是否具体
- 是否有量化标准
- 是否覆盖主要场景

#### 实用性评估
- 是否有明确应用价值
- 是否解决实际问题
- 是否易于集成到现有流程

### 3. 关联分析

识别模式之间的以下关系：

#### 增强关系
- A模式的实施有助于B模式效果的增强
- 示例: 推理外显 + 层次化解释

#### 依赖关系
- A模式的实施需要以B模式为基础
- 示例: 置信度标注 + 溯源标注

#### 冲突关系
- A模式的实施可能影响B模式的效果
- 示例: 推理外显 + 信息密度控制

## 文献研究方法

### 1. 来源评估

对每个信息来源，评估其权威性：

| 等级 | 来源类型 | 可靠性 |
|------|----------|--------|
| 高 | 学术论文、权威机构报告 | 高 |
| 中 | 技术博客、行业报告 | 中 |
| 低 | 个人博客、社交媒体 | 低 |

### 2. 信息提取

从文献中提取以下信息：

- 核心概念和定义
- 代表性工作和工具
- 应用场景和案例
- 评估方法和指标

### 3. 关联映射

将研究发现与现有模式进行映射：

- 支持现有模式的理论
- 提示新模式的方向
- 改进现有模式的建议

## 模式设计方法

### 1. 新模式识别

从以下来源识别新模式：

- 文献研究中的新概念
- 分析中发现的改进空间
- 用户反馈中的需求

### 2. 模式定义

每个新模式必须包含：

1. **名称**: 使用"XX模式"格式
2. **分类**: 功能分类 + 层次分类
3. **目的**: 一句话描述
4. **描述**: 详细原理说明
5. **要素**: 核心构成要素
6. **场景**: 适用场景列表
7. **实施**: 具体实施步骤
8. **示例**: 正面和反面示例
9. **注意**: 注意事项
10. **关联**: 与其他模式的关系

### 3. 模式验证

新模式需要通过以下验证：

- 结构完整性检查
- 与现有模式的一致性检查
- 实际应用可行性评估

## 质量审查方法

### 1. 审查维度

| 维度 | 检查项 |
|------|--------|
| 结构 | 是否包含所有必要要素 |
| 内容 | 描述是否清晰准确 |
| 可操作性 | 实施步骤是否可执行 |
| 实用性 | 是否有应用价值 |
| 一致性 | 风格是否与基线一致 |

### 2. 审查等级

- **P0 必须修改**: 影响基本功能的缺陷
- **P1 建议修改**: 影响质量的次要问题
- **P2 可选修改**: 优化建议

### 3. 审查流程

1. 逐项检查各维度
2. 识别问题并分类
3. 提供具体修改建议
4. 给出最终结论
