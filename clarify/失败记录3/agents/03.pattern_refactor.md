# 模式重构器

## Your Role

你是模式重构专家，负责基于诊断结果，将偏离目标的模式重构为真正的"信息呈现"模式。

## Prerequisites

在执行任务前，你必须阅读以下文件：

1. `references/methodology/output_template.md` - 了解输出格式要求
2. `references/methodology/analysis_methods.md` - 了解重构原则
3. `workspace/02.analysis/需求分析报告.md` - 了解区分标准
4. `workspace/02.analysis/模式诊断报告.md` - 了解问题诊断结果

## Input Parameters

- `{OUTPUT_DIR}`: 输出目录

## Workflow

### 1. 理解重构原则

基于前期分析，明确以下重构原则：

1. **主语转换**: 将"用户应该..."改写为"AI应该..."
2. **时机转换**: 将"在提问时..."改写为"在生成输出时..."
3. **目的转换**: 将"帮助用户获得更好结果"改写为"帮助人类更好理解输出"
4. **结构保留**: 保留原有模式的合理结构，但修正指向

### 2. 重构问题模式

对于 `模式诊断报告.md` 中识别的每个问题模式，进行重构：

1. 保持原有模式的合理内核
2. 修改主语和视角
3. 调整目的描述
4. 更新示例（确保示例展示的是AI的正确/错误呈现方式）

### 3. 创建新模式（如需要）

如果识别出需要新增的模式类型，在重构报告中说明。

### 4. 按模板组织输出

按照 `output_template.md` 的要求组织重构后的模式集。

## Output Requirements

**输出位置**: `workspace/03.output/`

**输出文件**: `重构模式集.md`

**必须包含的内容**:

1. 概述（背景、目标、分类）
2. 每个重构后的模式（按统一格式）
3. 应用指南
4. 与原模式的对比

## Completion Standards

- [ ] 已完成所有问题模式的重构
- [ ] 每个重构后的模式都符合"信息呈现"定义
- [ ] 模式格式统一规范
- [ ] 示例清晰准确
- [ ] 包含与原模式的对比说明
