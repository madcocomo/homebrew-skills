# 分析方法指南

## 分析框架

本研究采用"问题诊断-概念澄清-模式重构"三阶段分析框架。

### 阶段1: 问题诊断

**目标**: 识别已有模式偏离"信息呈现"目标的具体表现

**分析方法**:

1. **主语分析法**: 检查每个模式的描述主语
   - 如果主语是"用户"、"你应该..."、"在提问时..."，则属于提示词优化
   - 如果主语是"AI"、"系统"、"当...时应该..."，则属于信息呈现

2. **目的分析法**: 检查每个模式的最终目的
   - 如果目的是"让用户获得更好的输出"，则属于提示词优化
   - 如果目的是"让输出更容易被理解"，则属于信息呈现

3. **时机分析法**: 检查模式应用的时机
   - 如果是在用户提问时应用，则属于提示词优化
   - 如果是在AI生成输出时应用，则属于信息呈现

### 阶段2: 概念澄清

**目标**: 明确"信息呈现"模式的定义和边界

**核心问题**:

1. 信息呈现模式的**必要条件**是什么？
2. 信息呈现模式与提示词优化模式的**核心区别**是什么？
3. 一个好的信息呈现模式应该包含哪些**结构化要素**？

### 阶段3: 模式重构

**目标**: 将偏离目标的模式重构为真正的信息呈现模式

**重构原则**:

1. **主语转换**: 将"用户应该..."改写为"AI应该..."
2. **时机转换**: 将"在提问时..."改写为"在生成输出时..."
3. **目的转换**: 将"帮助用户获得更好结果"改写为"帮助人类更好理解输出"
4. **结构保留**: 保留原有模式的合理结构（目的、描述、示例等），但修正指向

## 分析输出格式

### 模式诊断报告格式

```markdown
## [模式名称] 诊断报告

### 原描述
[原始模式内容]

### 问题类型
- [ ] 提示词优化倾向
- [ ] 主语错误
- [ ] 时机错误
- [ ] 目的偏离

### 具体问题
[详细说明问题所在]

### 重构建议
[具体的重构方向]
```

### 重构模式格式

```markdown
## [模式名称]

### 目的
[简明说明此模式的目的，聚焦于"帮助人类理解"]

### 描述
[详细描述AI应该如何呈现信息]

### 呈现格式
[具体的输出格式规范]

### 示例

#### 正确示例
[展示正确的呈现方式]

#### 错误示例（呈现层面的错误）
[展示错误的信息呈现方式，而非"用户没指导AI如何做"]

### 适用场景
[明确此模式应在什么场景下使用]

### 与其他模式的关系
[说明此模式与其他呈现模式的关系]
```

## 质量检查清单

在完成每个模式的分析后，检查以下问题：

- [ ] 模式描述的主语是否是"AI"或"系统"？
- [ ] 模式是否聚焦于输出的结构和格式？
- [ ] 模式的目的是否是帮助"人类理解"而非"AI改进"？
- [ ] 示例是否展示的是"AI应该如何呈现"而非"用户应该如何提问"？
- [ ] 模式的适用场景是否清晰？
