# 代码重构工作流

重构优化现有 `avia-base` 或 `acorn` 代码。

## 重构原则

1. **保持 API 行为不变**
2. **小步修改，频繁测试**
3. **遵循项目代码风格**

## 重构步骤

### 1. 分析现有代码

- 理解当前实现和依赖关系
- 识别要重构的具体代码段
- 确认现有测试覆盖范围

### 2. 确保测试覆盖

- 如果测试不充分，先补充测试
- 运行现有测试确认通过

### 3. 执行重构

- 小步修改，频繁提交
- 保持 API 行为不变
- 遵循项目代码风格

### 4. 验证无回归

```bash
cd avia-base && mvn test
```

## 常见重构类型

| 类型 | 示例 |
|------|------|
| 抽取工具类 | 将路径解析逻辑抽取到 `PathUtil` |
| 简化 Service | 拆分大 Service 为多个小 Service |
| 统一错误处理 | 抽取异常处理到 `@RestControllerAdvice` |
| 消除重复 | 抽取公共方法到基类或工具类 |

## 检查清单

- [ ] 重构后代码更简洁
- [ ] API 行为保持不变
- [ ] 所有测试通过
- [ ] 代码风格一致
- [ ] 无硬编码值

## 常用命令

```bash
# 运行测试
cd avia-base && mvn test

# 运行特定测试
mvn test -Dcucumber.filter.name=".*场景名.*"

# 检查代码风格（如果有）
mvn checkstyle:check
```
